# WaEnhancer Theme System

This document explains the Material3 (Material Design 3) theme implementation in WaEnhancer, including support for light/dark modes, automatic system theme following, and dynamic colors.

## Theme Mode Selection

WaEnhancer supports three theme modes accessible from **General > Dark Theme Wa Enhancer**:

| Mode | Value | Description |
|------|-------|-------------|
| Follow System | 0 | Theme automatically follows the Android system dark mode setting |
| Dark Theme On | 1 | Always uses dark theme regardless of system setting |
| Dark Theme Off | 2 | Always uses light theme regardless of system setting |

## Material3 Implementation

### Theme Hierarchy

The theme system is built on Material3 with the following hierarchy:

```
Light Mode:
AppTheme (values/styles.xml)
  └── Theme.Light (values/themes.xml)
      └── Base.AppTheme.Light
          └── Theme.Material3.DynamicColors.Light.Rikka (with dynamic color support)

Dark Mode:
AppTheme (values-night/theme.xml)
  └── Theme (values/themes.xml)
      └── Base.AppTheme
          └── Theme.Material3.DynamicColors.Dark.Rikka (with dynamic color support)
```

### Theme Overlays

Additional theme overlays are applied in `BaseActivity.onCreate()`:

1. **Rikka Material3 Preference** - Styles preference screens with Material3 components
2. **ThemeOverlay** - Custom preference styling (removes dividers, etc.)
3. **ThemeOverlay.MaterialGreen** - Green accent color generated by Material Theme Builder

### Key Files

| File | Purpose |
|------|---------|
| `res/values/themes.xml` | Base light and dark themes with Material3 |
| `res/values-night/theme.xml` | Night mode theme inheritance |
| `res/values/styles.xml` | Day mode AppTheme and widget styles |
| `res/values/theme_customs.xml` | Light mode MaterialGreen overlay |
| `App.java` | Theme mode switching via AppCompatDelegate |
| `BaseActivity.java` | Theme application and overlay stacking |
| `BasePreferenceFragment.java` | Theme preference change handling |

## How Theme Switching Works

1. **User selects theme mode** in preferences (`thememode` key)
2. **BasePreferenceFragment** detects the change in `onSharedPreferenceChanged()`
3. **App.setThemeMode()** is called with the new mode value
4. **AppCompatDelegate.setDefaultNightMode()** updates the system-wide mode:
   - `MODE_NIGHT_FOLLOW_SYSTEM` (mode 0)
   - `MODE_NIGHT_YES` (mode 1)
   - `MODE_NIGHT_NO` (mode 2)
5. **Activities recreate** automatically to apply the new theme
6. **Android resource qualifiers** (`-night`) select the correct resources

## Android Version Compatibility

### Android 10+ (API 29)
- Full support for system dark mode following
- `MODE_NIGHT_FOLLOW_SYSTEM` works with system settings

### Android 12+ (API 31)
- **Dynamic Colors (Material You)**: Theme colors automatically adapt to wallpaper
- The `Theme.Material3.DynamicColors.*.Rikka` themes enable this automatically
- MaterialGreen overlay provides fallback accent color

### Android 16+
- All Material3 features supported
- Enhanced predictive back animations
- Improved theme transitions

## Build Configuration

The `materialThemeBuilder` plugin in `app/build.gradle.kts` generates the MaterialGreen theme overlays:

```kotlin
materialThemeBuilder {
    themes {
        create("MaterialGreen") {
            lightThemeFormat = "ThemeOverlay.Light.%s"
            darkThemeFormat = "ThemeOverlay.Dark.%s"
            primaryColor = "#4FAF50"
        }
    }
    generatePalette = true  // Enables Material3 color tokens
}
```

## Dependencies

The theme system relies on:

- `com.google.android.material:material` - Material Components for Android
- `dev.rikka.rikkax.material:material` - Rikka Material extensions
- `dev.rikka.rikkax.material:material-preference` - Material3 preference styling
- `dev.rikka.tools.materialthemebuilder` - Theme generation plugin

## Best Practices for Developers

### Adding New Activities

Extend `BaseActivity` to automatically get theme support:

```java
public class MyActivity extends BaseActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        // Theme is already applied
    }
}
```

### Using Theme Colors

Use theme attributes instead of hardcoded colors:

```xml
<!-- Good: Uses theme color -->
<TextView android:textColor="?colorPrimary" />

<!-- Avoid: Hardcoded color -->
<TextView android:textColor="#4FAF50" />
```

### Creating Theme-Aware Resources

Create variants in both `values/` and `values-night/`:

```
res/values/colors.xml         <!-- Light mode colors -->
res/values-night/colors.xml   <!-- Dark mode colors -->
```

## Xposed Hook Compatibility

The theme system is designed to not interfere with WaEnhancer's Xposed hooks:

- Theme logic is isolated in the `activities.base` package
- No hooks modify theme-related system components
- CSS theme injection (WhatsApp theming) is separate from app theming
